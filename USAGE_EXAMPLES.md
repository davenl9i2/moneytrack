# 使用範例

## 💬 對話範例

### 範例 1: 基本記帳
```
使用者: 午餐 100
Bot: ✅ 記帳成功！
     類型: 支出
     項目: 飲食
     金額: -100
     備註: 無
```

### 範例 2: 查詢昨日支出
```
使用者: 我昨天花多少錢?
Bot: 📊 查詢結果

     支出總額: $1,250
     筆數: 8

     主要分類:
     飲食: $600
     交通: $350
     購物: $300
```

### 範例 3: 查詢特定分類
```
使用者: 這個月的交通費
Bot: 📊 查詢結果

     支出總額: $2,100
     筆數: 15

     主要分類:
     交通: $2,100
```

### 範例 4: 查詢收入
```
使用者: 本月收入多少?
Bot: 📊 查詢結果

     收入總額: $50,000
     筆數: 2

     主要分類:
     收入: $45,000
     其他: $5,000
```

### 範例 5: 一般對話
```
使用者: 你好
Bot: 您好！我是您的記帳小幫手 😊
```

## 🎯 支援的查詢類型

### 時間範圍
- ✅ 昨天
- ✅ 今天
- ✅ 這週 / 本週
- ✅ 上週
- ✅ 這個月 / 本月
- ✅ 上個月

### 查詢類型
- ✅ 支出 (預設)
- ✅ 收入
- ✅ 全部收支

### 分類篩選
- ✅ 飲食
- ✅ 交通
- ✅ 購物
- ✅ 娛樂
- ✅ 其他

## 🔄 混合使用

您可以在同一個對話中混合使用記帳和查詢：

```
使用者: 早餐 80
Bot: ✅ 記帳成功！...

使用者: 我今天花了多少?
Bot: 📊 查詢結果
     支出總額: $80
     筆數: 1
     主要分類:
     飲食: $80

使用者: 午餐 120
Bot: ✅ 記帳成功！...

使用者: 我今天花了多少?
Bot: 📊 查詢結果
     支出總額: $200
     筆數: 2
     主要分類:
     飲食: $200
```

## 🚀 提示與技巧

1. **自然語言**: 不需要特定格式，用自然的方式提問即可
2. **中英文混用**: 支援中英文混合輸入
3. **即時更新**: 查詢結果會即時反映最新的記帳資料
4. **智能分類**: LLM 會自動判斷最適合的分類
5. **容錯處理**: 如果無法理解，Bot 會提供範例說明
